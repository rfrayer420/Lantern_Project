## Folder Roles
- `Canon_Core/` — lore, continuity, state indexes, canon rules.
- `Visual_References/` — character image references.
- `Bestiary/` — monsters, stats, weaknesses, dice mechanics, and creature PDFs.
- `Novels/` — manuscripts and prose.
- `Exports/` — finished deliverables.
- `Games/` — game rules, mechanics, assets.
- `Metadata/` — character metadata, changelogs, indexes.

## Visual Canon Workflow
For each character:
1. Existing images are stored in `Visual_References/<Character>/retired/`.
2. Generate/refine one definitive approved character image.
3. After George approves it, save it as `Visual_References/<Character>/<Character>_Canon.png`.
4. Future image generation must use only that approved canon image for that character.
5. Old images remain retired and must not be used as active references unless George explicitly reactivates them.
6. Update `LIVE_CANON_INDEX.md` after each approval.
